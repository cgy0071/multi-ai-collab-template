# 首个 Commit 建议

## 推荐 commit message

### 英文优先

```text
feat: initial public release of multi-AI collaboration template
```

### 更具体一点

```text
feat: add reusable multi-AI collaboration skills and handoff templates
```

### 如果你想强调 skill

```text
feat: publish lead/collaborator AI workflow skills and review templates
```

---

## 首个 commit 应该包含什么

建议首个 commit 一次性包含：

- 仓库 README
- 中文 README
- LICENSE
- CONTRIBUTING
- 安装说明
- skill 使用说明
- 角色 skill
- 各类协作文档模板
- 示例目录

这样首个 commit 就是一个完整可用的版本。

---

## 不建议放进首个 commit 的内容

- 大量后续想法
- 未验证的扩展模板
- 某个项目的 task/result/review 样例
- 与私有工程强绑定的流程说明

---

## 推荐首个 Tag

```text
v0.1.0
```

原因：

- 已经可用
- 但还明显会继续演进
- 适合定义为首个公开模板版本
