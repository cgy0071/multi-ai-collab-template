# Task Pack Format

Use this structure for collaborator-facing task packs.

## Required sections

1. `标题`
2. `背景`
3. `总体目标`
4. `关键约束`
5. `Task Pack A/B/C`
6. `停止条件`
7. `审核口径`
8. `推荐一句话指令`

## For each task pack item include

- `名称`
- `优先级`
- `Input`
- `任务内容`
- `Expected Output`
- `Validation Method`
- `Done Criteria`

## Good task pack properties

- one main question per round
- explicit allowed files
- explicit forbidden files
- explicit stop conditions
- direct mapping from task to evidence

## Bad task pack properties

- “顺手优化”
- no validation method
- no rollback condition
- mixed architecture + cleanup + infra in one bundle
