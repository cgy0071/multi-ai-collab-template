# Baseline Rules

Formal baseline updates must be conservative.

## Promote only when

1. evidence is concrete
2. the test scope is labeled
3. the result is reviewed
4. the result is not contradicted by newer evidence

## Always label

- tool version
- dataset or workload
- sample count
- start index or seed
- test type

## Never promote directly

- raw collaborator guesses
- unreviewed results
- mixed-scope experiments with unclear attribution
- generated cache or temporary logs as if they were formal docs
