# Result Format

Use this structure for collaborator outputs.

## Sections

1. `任务名称`
2. `输入文件`
3. `实际执行内容`
4. `关键证据`
5. `结果`
6. `不确定点`
7. `建议下一步`

## Good result properties

- factual
- scoped
- evidence-backed
- explicit about uncertainty

## Bad result properties

- replacing the project manager’s review
- declaring formal baseline updates
- mixing execution facts with strategic decisions
