# Execution Rules

## Before acting

Confirm:

1. the main question of the round
2. allowed files
3. forbidden files
4. validation steps
5. stop conditions

## While acting

- keep the experiment attributable
- change one main variable per round when possible
- do not merge unrelated improvements

## After acting

Always report:

- what changed
- what was validated
- what failed
- what remains uncertain

## When blocked

Return a blocked result instead of improvising a new project direction.
